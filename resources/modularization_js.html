Modularization

Introduction

Have you ever noticed that when you visit a website and move from page to page that there are certain areas on the page that don't change. To see what I mean let's visit subway.com (http://subway.com). Click around in the subway site. On each page scroll up and down and look for those parts of each page that don't really change. What did you find? If you said the top (header) and bottom (footer) you are very observant. These common parts are highlighted in figure 1:

Benefits

It is these common parts of a web page that can be reused. We call snippets of code that make up these reusable sections of the web page "modules" and the process of implementing them into the page "modularization". The benefits of modularizing our websites is to:

  • Save time - when we build modules of common code we only have to do it once. This means less time writing the same code again and again.
  • Easy maintenance - By having this common code stored off in a module, when it comes time to make a change to existing code, add new code or delete code, we only have to do it once for it to take effect throughout all the pages that use it.
  • Save Money - whether you're building a web site for yourself or someone else, the less time you have to spend building and maintaining it means less time and expense doing so. In business terms, it is being efficient.

Modules

You may be inclined to ask, "What exactly is a module?" The answer is, "A small amount of code, not stand-alone, that can be plugged-in to a larger collection of code in order to carry out or serve a particular task". Going back to Subway.com's header, you will notice that on the left is the logo, in the top right corner is a link to their international web site, to the left of the link are various tools, and at the bottom of the header is the main navigation bar. If you were to look at the source code for any of the pages you would find that the code used to build the header is about 100 lines long (alot of that is for the drop-down menus). But if you were to copy that code and paste it on a page of its own, it would look pretty strange. It is not meant to be used in that way, it is meant to be used at the top of each web page to add specific functionality to the page as a whole.

A simplified header example might lead us to create a code snippet that looks like this:

     <a href="/" title="Go to the subway.com home page">
          <img src="/images/subway-logo.jpeg" alt="subway.com logo">
     </a>
     <div id="tools"> ... code for various tools goes here ... </div>
     <nav id="page-nav">
          <ul>
               <li><a href="/">Home</a><li>
               <li><a href="/sandwiches">Sandwiches</a><li>
               <li><a href="/ingredients">Ingredients</a><li>
               <li><a href="/catering">Catering</a><li>
               <li><a href="/nutrition">Nutrition</a><li>
               <li><a href="/locations">Locations</a><li>
               <li><a href="/careers">Careers</a><li>
               <li><a href="/about">About Subway</a><li>
          </ul>
     </nav>

This code would be saved into a new text file, nothing else in the file except this code and given an appropriate name (e.g. header.html). The header.html file would be stored in a folder with other modules, so the storage folder could be named "modules" or something similar.

An explanation of the code:

  • <script src="https://code.jquery.com/jquery-3.1.0.min.js" integrity="sha256-cCueBR6CsyA4/9szpPfrX3s49M9vUU5BgtiJj06wt/s=" crossorigin="anonymous"></script> - this is a request to get the jQuery library directly from the jquery.com web site.   • <script src="modules.js"></script> - this is a request to our own JavaScript file, stored in our own website.
  • $(function() {
    $("#page-header").load("header.html");
    });     - This is an anonymous function that runs when the rest of the html document is finished loading. It directs the contents of the header.html file to be loaded into the container with an id of "page-header".

The Result

Since JavaScript runs in the browser, here is how all of this code would work. Someone would see a link to index.html and click it. The server would deliver the index.html file to the browser. When the browser gets to the <script> tags it would read the files referenced in the "src" attributes. Reading the modules.js file, the function would be run, and the contents of the header.html file would be inserted into the <header> element. Assuming that everything worked as it was supposed to, the header would look like this when the browser was done:

Make a Change

Reflect back on the list of benefits of modularization. Let's suppose that the folks at Subway decide to make soup a big part of their menu. Wow, we need to update that list of items in the navigation list. No problem. You open the header.html module and add the following code to the unordered list: <li><a href="/soup">Soups</a></li>.

Save the file and upload it to your website. Viola! You now have a soup navigation menu item throughout your website - even if the site had 10,000 pages. Pretty Cool, huh?!

Summary

Modularization is a great technique for writing and storing small collections of code that represent reusable portions of a website into external files. Those files are then included using JavaScript or some other scripting language. The resulting file is then delivered to the requesting user-agent (usually a web browser) in its finished form, leaving no evidence of the scripting language. The key is to use modules with those portions of a webpage that stay the same from page to page to page. Using this technique saves time in both construction and maintenance and money.

Application

As you prepare to build web pages, it is helpful to build a template(s). A template is simply a standard page (without content) that can be reused when a new content page is needed. The template(s) should include semantic structures with modularized common content as discussed in this activity. Use the steps outlined in this article to do so. Then, be sure to test it on the actual web server to insure that things are working as they should.
