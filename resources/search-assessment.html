Search Tool Assessment

Overview

Last week you built a static link that allowed you to request a JSON object from the wunderground web site for each of the cities we have worked with during the semester. Then, you were to break apart the JSON object and display the data into a web page.

This week, you have built a search tool to dynamically create a list of locations that match the term being typed into a search bar and created a link for each of those terms that, when clicked, would display the weather information for the location on the Weather Underground web site. However, by leaving your site your site visitors lose interest in what you have to offer.

This assessment will have you put the skills and concepts from both week's to work.

The Assessment

  1. You will return to your weather pages (which you began working with back in week 5) and add a "home" page.
  2. "Home" will also need to be added as a working link to the navigation bar.
  3. The home page will contain a search bar and, just like this week's activity, dynamically generate a list of locations based on the search term being typed.
    • Do NOT use the same search.js file as for the activity, make a new one and use it.
    • You may want to use CSS or JavaScript to show the search results until a link within it is clicked, then hide it.
  4. Replace the link built in the activity with one that will request a JSON object of weather data for the listed location (just like you did last week).
  5. When returned, the JSON object will be used to display the current weather data for the location in the "home" page.
  6. Thoroughly test your work to ensure it works prior to the submission deadline.

Collaboration

  • Work in groups of 2 to 5
  • Help one another with the assessment
  • Add a comment between the </head> and <body> tags of the home page with a title of "Assessment Collaboration Team" and then list the names of each person you worked with in an ordered list format. Put your own name first.
  • Remember the list is in the code as a comment and should not be visible in the browser.

Submit Your Work

Post the new home page to your site, along with any other updated files in the weather site and create a link to the home page on your temporary home page with the link text of "Weather Home".

Go to this week's collaboration board and post the total amount of time needed to complete all work (readings, activity and assessment) for the week.

Grading Matrix

Objective 5 (45 points)

  1. The search bar on the home page dynamically generates a list of locations that match the search term when javaScript is enabled. - 3 points
  2. Navigation bar links to the Franklin, Greenville and Springfield pages load the respective weather data for all three cities into the home page and do not actually navigate away from it when javaScript is enabled. - 20 points
  3. With JavaScript disabled, the index page does not load any weather data. - 2 points
  4. With JavaScript disabled, navigation links navigate to the respective weather page and the weather data for that page is loaded. - 15 points (5 points per city page)
  5. View the "remote_wu_bp.js" file to confirm the onclick event handler is present and appears viable. - 5 points

Objective 6 (15 points)

  1. View the source code for the home page, the names of collaborators is shown for the assessment. - 5 points
  2. View the source code for the activity page (a separate link on the temporary home page), the names of collaborators is shown for the activity. - 5 points
  3. The time needed to complete all work was reported to this week's collaboration board. - 5 points
