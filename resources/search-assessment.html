Search Tool Assessment

Overview

Last week you built a link that was used to request a JSON object from the wunderground web site for the current location. Then, you broke apart the JSON object and displayed the data into a web page.

This week, you have built a search tool to dynamically create a list of locations that match the term being typed into a search bar and created a link for each of those terms that, when clicked, would display the weather information for the location on the Weather Underground web site. However, by leaving your site your site visitors lose interest in what you have to offer.

This assessment will have you put the skills and concepts from both week's to work.

The Assessment

  1. You will return to your weather pages (which you began working with back in week 5) and create a "search" page.
  2. "Search" will also need to be added as a working link to the navigation bar or to a tool bar in the page header.
  3. The search page will contain a search bar and, just like this week's activity, dynamically generate a list of locations based on the search term being typed.
    • Do NOT use the same search.js file as for the activity, make a new one and use it.
    • You may want to use CSS or JavaScript to show the search results until a link within it is clicked, then hide it.
  4. Replace the link built in this week's activity with one that will request a JSON object of weather data for the listed location (similar to the current location link you did last week).
  5. When returned, the JSON object will be used to display the current weather data for the location in the "search" page.
  6. The title and content headings must also update to reflect the location for which the weather data is being displayed (just like last week).
  7. Thoroughly test your work to ensure it works both locally and on the GitHub server prior to the submission deadline.

Collaboration

  • Work in groups of 2 to 5
  • Help one another with the assessment
  • Add a comment between the </head> and <body> tags of the home page with a title of "Assessment Collaboration Team" and then list the names of each person you worked with in an ordered list format. Put your own name first.
  • Remember the list is in the code as a comment and should not be visible in the browser.

Submit Your Work

Post the new search page to your site, along with any other updated files in the weather site to your GitHub server.

Go to this week's collaboration board and post the total amount of time needed to complete all work (readings, activity and assessment) for the week.

Grading Matrix

Objective 5 (30 points)

  1. A search link exists in the weather site and is accessible from any page in the site which leads to the search page. - 2 points.
  2. The search page contains a search bar which dynamically generates a list of locations that match the search term. - 3 points
  3. Each item in the dynamic list is a working link. - 5 points
  4. When clicked, the link requests and receives back a JSON object of weather data for that location (visible in the console tab of the developer tools). - 5 points
  5. The weather data for the clicked location is displayed in the search page:
    • The title shows the name of the location. - 2 points
    • The content heading shows the name of the location. - 2 points
    • The current temp is displyed along with the degree symbol. - 4 points
    • The current date high and low temps are displayed with degree symbols. - 4 points
    • A summary of the current date's weather is displayed. - 3 points

Objective 6 (15 points)

  1. View the source code for the search page, the names of collaborators is shown for the assessment. - 5 points
  2. View the source code for the activity page (a separate link on the temporary home page), the names of collaborators is shown for the activity. - 5 points
  3. The time needed to complete all work was reported to this week's collaboration board. - 5 points
