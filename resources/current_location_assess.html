Get The Data

All code for this assessment will be written only in the remote_wu.js file, no other code will be allowed with the exception of custom CSS as explained in step 8.3 below!

  1. You will probably want to review the Weather Underground links provided in the Weather Underground API activity as they indicate How to request data and the Type of data that you will need.
  2. Use the jQuery $.ajax method which begins on line 24 and currently ends on line 34.
  3. Write all code within this method, add new lines within the method as need.
  4. Send the URL to the weather underground API requesting both the geolocation and conditions objects. Use the method that allows you to send in the latitude and longitude for the current location. Don't forget to include your API key as required in the URL.
    1. The latitude (lat) and longitude (long) were gotten for you on lines 9 and 10.
  5. The dataType needed will be "jsonp".
  6. Write a success function and store its parameter as "data".
  7. I strongly encourage you to use console.log often to see what you are getting back from the API and your own variables.
  8. Base Data:
    1. You are to get the city and state and combine them into a single location name separated by a comma and space. The name is to be displayed (using a jQuery selector) in the h1 element with an id of cityDisplay AND as the first item in the "title" element within the "head" element of the page (do not replace what's there, add this in front of what's already there).
    2. You are to get the current temperature and display it into the li with an id of currentTemp. Make sure you use JavaScript to round the number to the closest integer and display the degree symbol following the number.
    3. Finally, you are to display a summary of the current conditions in the li with an id of summary. Be sure the text starts with a capital letter (Hint: use the weather property from the current_observation object).
  9. Additional Data:
    1. You are to add a minimum of two additional data points to the weather page.
      1. Three list items are provided in the index page, each with an id of "add1", "add2" and "add3" for your use.
      2. Be sure that whatever you add makes sense from the site visitors perspective, don't make them guess what you meant - make it clear and obvious!
      3. You may alter the CSS within the user_styles.css file to style this content if needed (this file is already set to be used, you don't need to do anything with the main stylesheet).

Grading Matrix

Objective 5 (25 points)

The data is successfully returned and displayed in the web page, using:

  • Only the ajax method found in the remote_wu.js file.  - 5 points
  • The three base data points are present:
         The location name is the city and state, separated with a comma and space - 3 points
         The location name appears in the content "h1" and as the first value in the "title" element - 2 points
         The temperature includes the degree symbol - 2 point
  • The summary value starts with a capital letter - 2 point
  • A minimum of two additional data points are included. - 6 points
  • The additional data points are clearly legible and communicate the information unambiguously. - 5 points

The page comment lists the other class members that the developer collaborated with during the assessment.
