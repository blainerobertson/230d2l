Responsive Design

Last week you built a web page to display weather data for the city of Franklin. In addition, you wrote CSS to add visual appeal to the web page. But, all of the markup and styling was specific to a small screen device. This week you are to add CSS3 Media Queries to isolate the styling for medium and large screen devices.

Mobile First

The default styling that exists outside of media queries should accomplish two tasks:

  1. Provide default styling for all screen sizes (font type, color scheme, etc...).
  2. Provide specific layout styling for small screens.

Adapting Layout and Styling for Larger Screens

You should write a minimum of two media queries:

  1. Medium screens - typically these would be tablet type devices
  2. Large screens - typically these would be laptop / desktop devices

However, while not required for this assessment, you may want to think about and possibly play with very large screens (4k and 5k monitors). These high density screens provide their own challenges for displaying content.

Use Your Wireframe Sketches

In week 4 you were to produce 3 wireframe sketches. This week you are to add media queries (mentioned above) and then add CSS code to the media queries to cause the Franklin weather data to display in all screen sizes according to the layout shown in your wireframe sketches.

The wireframe sketches for all three devices must be included at the bottom of the Franklin weather page (just above the page footer) as reference when the page is being reviewed for matching your design plan.

Include the ViewPort Meta Element

In case you missed it in this week's readings, make sure to read about the viewport meta element and include it in the head section of the Franklin web page.

Add the Hourly Temperature Data

Add the hourly weather data for the city of Franklin to the Franklin weather page from the weather.json file. But, only display the data on medium or large screen devices.

Submission

The revised Franklin weather page and CSS stylesheet used to style the Franklin page will replace the files from last week. They should be accessed using the same link that should already exist in your temporary home page. Be sure to test that the new versions appear once the files are uploaded to your web site. It is recommended that you test on a variety of devices.

Grading Matrix

Objectives 1 and 2 are assessed in this week's work.

Objective 1 (15 points)

  1. The page has a meaningful, content-related title. - 2 points
  2. The file name follows the web naming rules. - 2 points
  3. The viewport meta element is present in the Franklin weather page. - 2 points
  4. The Franklin weather page is constructed using semantic markup (e.g. header, nav, main, footer). - 2 points
  5. The Franklin weather page is valid HTML5. - 4 points
  6. Wireframe sketches for all screen sizes (small, medium and large) are included in the Franklin weather page. - 3 points

Objective 2 (20 points)

  1. A minimum of two media queries are included in the same external CSS page as built for the Franklin weather page last week (1 for medium and 1 for large screens). - 4 points (2 points each)
  2. Additional CSS code has been added to the media queries to change the layout of the weather data to match the wireframe sketches. - 4 points (2 points for each code implementation)
  3. The weather data is easily read and visually appropriate to the size of the screen. - 2 points (Yes or No)
  4. The PARC and typography design principles are evident in the visual appearance of the Franklin weather page. - 3 points (1 point will be deducted for each missing design principle)
  5. The hourly temperature data is only visible in medium and large screen devices. - 4 points
  6. The CSS is valid CSS3. - 3 points

You are strongly encouraged to trade grading your assessment with a classmate before the due date in order to catch and fix errors prior to it being graded by the professor or teaching assistant.
