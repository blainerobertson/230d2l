Implement Responsive Images

Assumptions

Prior to doing this activity it is assumed that you have:

  1. Read the assigned readings for this week listed in the overview page:
    • Web Image Types
    • Multiple Background Images
    • Using srcset and sizes
    • Using srcset
  2. Watched the Responsive Images video (SitePoint Premium)
  3. Downloaded the responsive images zip file
  4. Completed the Create Responsive Images activity
  5. Completed the Optimize Images activity

If you have not done these items, as listed, please do so before proceeding with this activity.

Work Together

All through this course you have been encouraged and sometimes required to work with others. You are strongly encouraged to work with at least one other member of the class on this activity!

The Activity

This is an application activity, which is to find out if you can apply what you understand from the preparatory readings and video. Begin by reviewing the index.html file found in the downloaded zip file. The markup for the page is complete and should not be changed except to add the code needed for the responsive content image as directed below.

The Page Header

The index page has a page header, specified using the markup "<header class="page-header">". You will find a corresponding class in the style.css file in the css folder. There you will find three selectors already written and waiting for you - one for small screens on line 52, one for medium screens on line 173 and one for large screens on line 218.

You are to write css code to enable multiple background images into the header. The banner image you created and optimized should be used as well as the nps-logo that you created and optimized. The smallest versions of these files should be used for small screens, the medium versions for medium screens and the large versions for the large screens. Each of these respective usages will occur in the rule for that size of a screen.

To implement these multiple backgrounds, the html should not be changed at all. All implementation will take place in the style.css file. When done, your page header should look similar to the image below:

The Content Image

The index.html file has a content image placeholder. The placeholder is "<figure class="featured-image">" and within it is an <img> element. All markup for this activity will occur in or around the <img> element in the index.html page. There should be no changes made to the css styles for either of these elements.

You are to use the "srcset" alone, or the "srcset" and "sizes" attributes in combination within the <img> element, or replace the <img> element and replace it with a <picture> element in order to implement the responsive images.

The outcome is to deliver the smallest version of your optimized images (from the Creating Images and Optimizing Images) to the screen depending on screen size, viewport width or pixel density of the screen. Based on your readings and video demonstrations there are multiple possible solutions.

If your images used "art direction" to crop in and focus on a particular aspect of an image as the image became smaller (which you should have done), then it should be easy to see that the images are changing based on either screen size, viewport width or device pixel density.
