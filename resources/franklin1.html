Mobile Weather Page

Last week you produced some sketches of a web page related to weather. This week you will build the web page for the mobile screen as you sketched it. I strongly encourage you to work together to do this, so you can share understanding, ideas and ask questions and solicit feedback during the construction.

Your web page should do the following:

  • Be constructed entirely in HTML
  • The HTML should use semantic, structural elements
  • The HTML5 should be valid
  • Focus entirely on its rendering on mobile screens
  • Include all temperature-related data (except for the hourly temperatures) for the city of Franklin from the weather.json file
  • Include an image that is related to the weather summary statement found in the weather.json file for Franklin
  • Include navigation items (they do not need to work) for all three cities found in the weather.json file
  • Include the mobile sketch for the weather page that you produced last week.
  • Create a link to the external stylesheet in the <head> of this page to allow the CSS to affect the appearance of the html page.

Styling the Weather Page

Using CSS

This week has been a whirlwind week of jumping into CSS. You have read a chapter and watched (and hopefully followed along) with a video course on how CSS works and is applied to a web page.

The Mobile Web Page

Prior to undertaking the task of styling the mobile web page and the content in it, you must first have built it. Make sure the is already done before starting this styling task.

Styling the Page

We are starting pretty basic with this styling task. No need to get fancy yet. Your primary tasks are listed below:

  • Make good use of the web design principles (PARC and Typography) covered previously.
  • Implement some of the techniques from the good mobile examples that you wrote about to guide what you do.
  • The styling should follow your sketch for mobile size screens from last week.
  • Use an external CSS file, that you create and write the code for, to style the mobile weather page.
  • Ensure that you use appropriate font sizes and colors to allow easy readability.
  • Use colors to enhance the message of a sunny day in Franklin and that work well with the image.
  • The navigation items are easy to identify and reach, but unobtrusive.
  • The CSS must be valid CSS level 3.

Submitting

  • The weather page will be uploaded to your remote hosting site.
  • Add a link to the weather page in your temporary home page.
  • The stylesheet should be linked to the weather page using a <link> element in the <head> of the weather page.
  • The stylesheet, weather page and all images must be uploaded to the remote server.
  • Once everything is uploaded, open your site using the 230 Student Site List and test everything to make sure it is working. The testing should also be done using a mobile device!
  • If things do not work, troubleshoot and work with it until it does. Get help if needed.

Grading Matrix

This assessment is worth 14 points for Objective 1.

  • The page has a meaningful content-related title - 1 point
  • The page is named following the web naming rules.
  • The page uses semantic structural elements - 2 points
  • The page is valid HTML5 - 2 points
  • The Franklin data is present and contained in html content elements - 2 points
  • There is an image that is reflective of "Sunny" conditions in Franklin - 2 points
  • Navigation items are present for all three cities listed in the weather.json file (nav items do not need to work) - 2 points
  • The mobile sketch from last week is included as the last item in the weather page. - 2 points

This assessment is worth 12 points for Objective 2.

  • The page layout renders in a mobile size screen without requiring zooming - 1 point
  • The page layout renders in a mobile screen without any horizontal scrolling - 1 point
  • The PARC design principles are clearly reflected in the style of the screen - 2 points
  • The page is valid CSS3 - 2 points
  • The Franklin data is easily read (typography and size) - 2 points
  • The screen layout matches the sketch from the previous week - 2 points
  • Navigation items are styled to fit in a mobile screen without interfering with the data - 2 points
