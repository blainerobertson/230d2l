jQuery: Adding it to a web page

Version, versions, versions

  1. On the jQuery.com home page, find and click the large button to "Download jQuery".
  2. Scroll down until you find the heading for jQuery 2.x.
  3. You will find options "compressed, production jQuery 2.2.1" and "uncompressed, development jQuery 2.2.1".
  4. What's the difference you may ask?
    1. The "compressed" version has had all comments and spaces removed and variables have been shortened. Doing these things makes the code practically unreadable to humans, but computers can read the file just fine. It also makes the file smaller (86kb).
    2. The "uncompressed" version maintains spaces, comments and full variable names to make it easier for humans to read. This adds to its weight however leaving a file that weighs 259kb.
  5. If you scroll down even further you will find a section entitled, "Using jQuery with a CDN".
    1. You may ask, "What's a CDN?"
    2. CDN stands for "Content Delivery Network". It is a network of servers scattered across the world that store files, including the jQuery library, and allow other computers to download those files when needed.
    3. Because CDN's are distributed, and serve files constantly, they are very fast and may be closer to your site's visitor than your own server is.
    4. This means that if you add code to your web page telling the browser to download jQuery from a CDN it will arrive faster and be lighter than what it would be if you put it on your own server.
  6. If you scroll down a bit further you will see that jQuery can be obtains from a number of CDNs:
    1. http://code.jquery.com - jQuery's own CDN
    2. https://developers.google.com/speed/libraries/devguide#jquery - Google's CDN
    3. http://www.asp.net/ajaxlibrary/cdn.ashx#jQuery_Releases_on_the_CDN_0 - Microsoft's CDN
    4. http://cdnjs.com/libraries/jquery/ - a CDN popular with JavaScript resources
    5. http://www.jsdelivr.com/#!jquery - another popular JavaScript-centric CDN
  7. There are two benefits to using a CDN:
    1. Typically the delivery of jQuery to the browser is faster
    2. The jQuery library is compressed as well as minified and thus weighs even less.

Size Comparison

I ran a test downloading JQuery version 2.2.1 (minified) from all CDN's listed on jQuery.com. Here is the size of the file as reported using YSlow:

  • Local delivery (non-CDN): 85.5K
  • jQuery CDN: 34.9K
  • Microsoft CDN: 38.4K
  • CDNJS CDN: 85.6K
  • Google CDN: 29.8K (version was 2.2.0)
  • jsDelivr CDN: 85.6K

Okay, which one do I use?

The answer is, "You want two". You want one that you can use for testing and development when you are not connected to the Internet - this will be the library that you will download and store on your own machine. You will want another one to actually use in your web pages, when they are being used in your hosting environment - this will be the copy gotten from the CDN.

Having said that, if you will always be connected to the Internet, always, then you could get by with one - the one from the CDN.

Alright then, Let's Do This

  1. Begin by creating a "javascript" or "js" folder in your htdocs folder (follow the web naming rules).
    1. Open Brackets, make sure you are working in the "htdocs" folder.
    2. In the left (Gray) panel, in empty space, right-click and select "New Folder".
    3. Name the new folder and press your "Enter" or "Return" key.
  2. In the new folder create a "cdn.txt" file.
    1. In Brackets, right-click the newly created folder.
    2. Click "New File".
    3. Name the file, "cdn.txt", press "Enter" or "Return".
    4. The file is named and a blinking cursor appears in the new, empty file.
  3. Return to the jQuery download page in your browser.
  4. Scroll until you find the heading "jQuery 2.x".
  5. Right-click the "Download the compressed, production jQuery 2.2.1" link and choose "Save link as...".
  6. When the dialog box opens, save the file to new folder created in step 1.
  7. Next, scroll down to the "Using jQuery with a CDN" heading.
  8. Choose one of the CDN's and click its link.
  9. Once there, look for jQuery version 2.2.1, with "min" in the name. (e.g. https://cdnjs.cloudflare.com/ajax/libs/jquery/2.2.1/jquery.min.js) and copy the whole line.
  10. Return to Brackets and paste the path into the cdn.txt file. Save the file.
  11. You now have the path that you will put into your web pages.

Getting jQuery into a Web Page

When the time comes you will want to add the code to insert jQuery into your web pages at the bottom of the page just before the closing body tag.

Example:


    </footer>
 </div><!-- Closes wrapper -->
<!-- Use for Production Server -->
 <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/2.2.1/jquery.min.js"></script>
 <!-- Use for Local Development -->
   <script src="js/jquery-2.2.1.min.js"></script>
</body>
</html>

Furthermore, you will want both paths present, but you will comment out one of them depending on if you are building and testing locally or uploading to the remote server.
