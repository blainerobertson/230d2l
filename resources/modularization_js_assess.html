Modularize the Weather Pages

This week you learned about modularization of basic, common code structures into the web pages. In this assessment you will implement common code modularization into both (Franklin and Greenville) weather pages.

Implement

When either of the weather pages is visited the common content (header, nav, footer) should be identical. In order to do this you will build the modules, use jQuery and write your own anonymous function using JavaScript as demonstrated in the modularization document and the embedded video. Make sure the implementation of modularization on these two pages has occurred.

Keep In Mind

This assessment is a trial run for your site project assessment that will occur at semester's end.

Grading Matrix

Objective 1 (5 points)

  1. The static content of the header, nav and footer have been physically removed from the Franklin and Greenville pages - 2 points
  2. The jQuery and JavaScript files are linked to the html pages using <script> elements - 2 points
  3. A <noscript> element is in place to allow graceful degradation if JavaScript is disabled - 1 point

Objective 4 (16 points)

  1. The page header is identical in both weather pages - 4 points (2 pts per page)
  2. The page navigation is identical in both weather pages - 4 points (2 points per page)
  3. The page footer is identical in both weather pages - 4 points (2 pts per page)
  4. The modularized content is fully operational - 4 points (2 pts per page)
