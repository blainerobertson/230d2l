Know Your CSS Selectors

As you watch the Introduction to jQuery video course and review the jQuery documentation, you are going to see "selectors" at work. A selector, as we have learned in CSS, is the means the browser uses to know what is being "selected" for styling or some other operation.

HTML

In HTML we have a variety of ways of marking up code:

  1. Using the element. All of the examples are simple opening tags of the element.
    For example, <p>, <nav>, <div>, <li>, <a>
  2. Adding a class to an element. Classes can be repeated on a web page!
    For example, <h1 class="page-title-heading">, <li class="box">, <li class="box"> (remember, they can be repeated)
  3. Adding an id to an element. There should only be ONE id with the same value per page.
    For example, <div id="wrapper">, <nav id="page-nav">, <footer id="page-footer">

CSS

In CSS, we use symbols to indicate if the CSS selector belongs to an element, class or id.

#page-nav - the # symbol indicates this is an "id" selector.

.page-title-heading - the period (.) indicates this is a "class" selector.

p - the element name, without the greater or less than symbols indicates an element selector.

In addition, remember that CSS can select based on the attribute of an html element:

a[href], img[src], li[class] - These are example of any element with the attribute in the brackets. There are also many ways to determine if the attribute possesses a value.

There are also children selectors (e.g. ul > li, header > nav). The element listed first is the parent, the ">" sign indicates that the element on the right must be a child.

The selectors listed are common, but there are many others. Refer to the list of CSS selectors found on W3Schools.com.

Why is this important?

jQuery uses the CSS selector syntax to identify the html that it too will interact with. You must remember this!
