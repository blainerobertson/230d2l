HTML5 Video

HTML5 Video Element

In the past, HTML relied on third-party tools such as Flash to place and run video inside of a web page. Fortunately, HTML5 and the modern browsers that support it make that a thing of the past. Now, browsers can support video natively, which makes things far more convenient for developers and site visitors. You can learn much more of the official information about HTML5 video at the W3C wiki reference at http://www.w3.org/wiki/HTML/Elements/video)!

In its simplest form the element looks like this:

<video src="path/filename"></video>

However, this wouldn't get us very far. So we add attributes to enhance functionality.

Attributes

Attribute Value(s) Description
src path and/or file name Indicate the location of the video file in the site structure.
controls None Boolean - Enables the default controls to play, pause, forward, reverse the playback as well as adjust volume
autoplay None Boolean - Enables the video to play automatically (don't do this!)
preload None, Auto, Metadata
  • None - do not load the file until play is activated.
  • Auto - download the file for playback as soon as the page loads. Including the preload attribute without a value is the same as preload="auto".
  • Metadata - download information about the file, but not the file itself.
loop None Boolean - does what it sounds like (don't do this).
poster path and/or file name Display an image representative of the video prior to the video being played.
height pixel value Sets the height of the video in the web page. Use CSS instead.
width pixel value Sets the width of the video in the web page. Use CSS instead.
muted None Boolean - Sets the audio of the video file, can override the user's preference.
type Mime type This provides the browser with additional information that it may need to be able to playback the file.

Including some common attributes, the audio element would more likely look like this:

<video controls preload="none" poster="path/filename" src="path/filename"></video>

Varied Format Support

In the past, different browsers supported different video formats, making it cumbersome for developers because instead of creating a single video, you would have to create three versions of the video file using different formats: MP4, OGV and WebM. Each type would have to be embedded in the video element using the <source> child element. See the example shown below:


<video controls preload="metadata" poster="/path/filename.png">
	<source src="/path/filename.mp4" type="video/mp4">
	<source src="/path/webm" type="video/webm">
	<source src="path/filename.ogv" type="video/ogg'>
	<p>Your browser does not support HTML5 video.
	You can <a href="/path/filename.zip"
	title="download a zipped version of the file">
	download a zipped version</a> (3 MB) of the
	file in MP4 format to your computer.</p>
</video>

Uniform Support

Thankfully, those days are now past. If you refer to the caniuse.com website you will see that virtually all modern browsers now support the MP4 format.

Converting Other Formats to MP4

Unfortunately, many programs and devices do not produce their video in the mp4 format. Instead they create video in ".mov" or ".wmv" formats. Converting either of these to MP4, is not hard, but necessary. Miro Video Converter is a free video conversion tool that is available for Windows and Mac - an important note, Miro VideoConverter is a conversion tool only, it is NOT a video editor, those tools are complex and are outside of the scope of this tutorial. Unfortunately, the tool is old, but still works and is simple to use. The tool can be downloaded from the Open Source Lab (sponsored by Oregon State University) or from my GitHub Repository using these links: Miro for Mac, Miro for Windows. In either case, version 3 of the tool is the most recent.

Create a Video File

You should work together with members of your learning group for the rest of this tutorial to help and support one another. When you are done converting your video file you will write the code to use it in the index.html page, along with the code for responsive images from the other activities within this responsive media lesson, then upload and test everything to make sure your media works in different browsers and on a variety of devices as well.

Prior to beginning you will need a video file, use this sample file (in zip format). Once downloaded, unzip the file and transfer the video to your responsive_media folder.

  1. Visit the Miro site or use one of the provided links above to download Miro Video converter. Once downloaded install MiroVideoConverter on your machine .
  2. Launch MiroVideoConverter and drag your video file to the drag and drop box in Miro as directed.
  3. From the drop down "Format" menu, choose "Video" and then "MP4".
  1. Click the "Convert to MP4" button and let the conversion complete.
  2. When done, click the "Show File" link, next to the video file image.
  3. Move the converted MP4 file from the current location to the location in your responsive_media folder where you want it stored.
  4. Check the file name. If more than one extension (such as two .mp4's at the end) exists, delete one of them leaving only a single .mp4 after the file name.

Test the Sample Video

To test the video simply drop it onto any browser icon in your computer or open a browser window and drag the video to the window. If the video begins playing you know that it is good to go. Easy, isn't it?!

Do It Again

Now that you have gone through the process with a sample video, let's do it again with a real video for the Grand Canyon page (index.html in the responsive_media folder).

In the raw-images folder you will find a video of mule pack trains used in the Grand Canyon, pack_mules.mov. This video is from the National Park Service and is in the public domain.

Convert it to mp4 format and save it to the images folder or if you want, create a "videos" folder and save it there. Then do the steps listed below.

Insert and Test the Video

  1. With the video file converted you are ready to implement and test.
  2. Use the index.html page in the responsive_media folder.
  3. In the web page, within the <main> element, find the <video> element already provided.
  4. Enter the "src" attribute and the correct path within the src attribute to point to the video file. Refer to the sample code provided in the Attributes section of this activity if needed.
  5. Test the web page and the video file in Live Preview and using different browsers and in different screen sizes.
  6. If things do not work as expected, troubleshoot to find and fix the error. Get help if necessary.
  7. Notice: When you are ready to upload the responsive_media folder to your remote hosting site be aware that video files can be very large! If the file weight is more that 1MB I recommend uploading the video files using a tool like FileZilla. It can handle the large file sizes, whereas programs like eqFTP for Brackets may fail in the uploading of very large files.
  8. Once uploaded to your remote hosting site, test the index.html file in the responsive_media folder with multiple browsers and on differing operating systems using multiple devices to make sure things work as planned.
