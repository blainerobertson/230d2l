Flexbox Implementation

Last week you added CSS Media Queries to your external stylesheet in order to affect the layout of the Franklin weather page on medium and large screen devices. This week you will take control of your page layouts for all three screen sizes using CSS Flexbox.

Use Your Wireframe Sketches

As always, the final layout for each device type will be judged against your wireframe sketches. If you have been provided feedback telling you that your sketches needed more detail to effectively judge where content should reside it is expected that your wireframe sketches will also have been updated.

The wireframe sketches for all three devices must be included at the bottom of the Franklin and Greenville weather pages (just above the page footer) as reference when the page is being reviewed for matching your design plan.

Mobile First

The default styling that exists outside of media queries should accomplish two tasks:

  1. Provide default styling for all screen sizes (font type, color scheme, etc...).
  2. Provide a specific layout for small screens.

Adapting Layout and Styling for Larger Screens

Your media queries should specifically change the layout and data presentation for medium and large screens (maybe even extra large screens). The layouts should now be controlled using CSS Flexbox.

Include Hourly Temperature Data

Add the hourly weather data to both city pages from the weather.json file. Make the hourly data accessible on all sizes of devices and if displayed, visible in an easily readable manner.

Navigation

The navigation links on both the Franklin and Greenville weather pages should now be operational for those two cities. The Springfield navigation item can remain non-operational.

Submission

The revised Franklin and new Greenville weather pages and CSS stylesheet used to style them will replace the files from last week. They should be accessed using the same weather link that should already exist on your temporary home page. Be sure to test the operation of the links and the weather pages in a variety of devices (or device emulators) once the files are uploaded to your web site.

It is strongly recommended that you exchange formative grading with other members of the class prior to the actual assessment in order to catch and correct problems. This exchange can be coordinated in-class or using the collaboration board.

Grading Matrix

Objectives 1, 2 and 4 are assessed in this week's work.

Objective 1 (20 points)

  1. The viewport meta element is present in both weather pages - 4 points (2 point per page)
  2. The Greenville page follows the web naming rules - 1 points
  3. The Greenville page has a meaningful, content related title - 2 points
  4. The Greenville page is constructed using semantic elements (including structural elements - header, nav, main, footer) - 2 points
  5. The hourly temperature data is included in the Greenville page - 4 points
  6. The Greenville page is valid HTML5 - 4 points
  7. Wireframe sketches for all screen sizes (small, medium and large) are included in the Greenville page - 3 points (1 point per sketch)

Objective 2 (30 points)

  1. Flexbox has been implemented in an external style sheet, used by both weather pages - 5 points (Yes or No)
  2. Flexbox changes occur within media queries for medium and large screened devices - 5 points (Yes or No)
  3. All weather data (including hourly temps) is easily read, visually appropriate and can be accessed in all screen sizes - 5 points (Yes or No)
  4. The PARC and typography design principles are evident in the visual appearance of both weather pages - 5 points (1 point will be deducted for each missing design principle)
  5. The CSS is valid CSS3 - 10 points

Objective 4 (24 points)

  1. Flexbox is fully operational and controls the layouts for small, medium and large screens. Both the Franklin and Greenville weather pages match the sketches for all three sized screens - 24 points (4 points for each screen size per page)
