Understand the Structure
Hopefully, over the past two weeks you have come to understand that curly braces "{}" are used to represent JavaScript objects.
 
What you may not have realized is that objects within object, such as:

{
    "Franklin": {
        "City": "Franklin",
        "State" : "ID",
        "High": 90,
        ...
    },

    "Greenville": {
        "City": "Greenville",
        "State" : "SC",
        "High": 84,
        ...
    },

    "Springfield": {
        "City": "Springfield",
        "State" : "IL",
        "High": 77,
        ...
    }
}

are "array-like". This means they are not true arrays, but they can be treated as if they were. For example, if our object were assigned to a variable:

var data = {"Franklin":{...},"Greenville":{...},"Springfield":{...}};

We could access all of the Greenville object's data by using data[Greenville], this mimics the array-like syntax.

If we were to use regular javaScript object syntax we could just use the regular dot notation (e.g. data.Greenville).

Mix and Match

But what is really cool is that you can (doesn't mean you should) mix and match syntax. For example, to get the high for Springfield you could use:

var high = data[Springfield].High;

or

var high = data.Springfield.High;

or assume that the value of Springfield had been stored in a variable, (e.g. var cityName), that too can be used in the array structure (but NOT with the dot notation structure):

var high = data[cityName].High;

Pretty cool, huh??!!!!
