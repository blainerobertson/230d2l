Bulletproof the Site

In this week's activities you have reviewed event handlers, JSON data and used PHP to get weather data for three city pages. Now, the assessment is to use event handlers to capture events from the links in the navigation menu, keep the default action from taking place, capturing data, feed that data to the API and show the resulting JSON data in the page. Whew, that is a lot of things.

Bulletproofing?

When you think of bulletproof you probably think of protective clothing that soldiers or police officers wear. While that is true, another definition is, "safe from failure; without errors or shortcomings and beyond criticism". (Dictionary.com)

That is what we are doing, making sure that if JavaScript is working (thus jQuery will work too) then the gathering of weather data will be done using jQuery. But, if JavaScript is not working (turned off), the gathering of data will continue to work (in this case for our city pages).

Write an Event Handler
All further work for this assessment will be done in the "remote_wu_bp.js" file. No other pages or code will be changed except as noted in the "Collaboration" section below. Using the resources mentioned in this and last week's activities, or any other resource, do the following:
  1. Write an event handler to detect when a link in the navigation bar is clicked.
  2. Prevent the default behavior of the link from occurring.
  3. Capture the text value of the link that was clicked and store it in a javaScript variable.
  4. Use console.log() frequently to see what is happening in the "developer console" of the browser.
  5. Write a jQuery AJAX function to read the data from the local weather.json file.
  6. Remember the dataType is "json"
  7. Upon a successful read of the data from the weather.json file capture the latitude and longitude for the city based upon which of the links was clicked on.
  8. Call the getData function  and pass in the longitude and latitude values captured from the local weather.json file.
  9. When you believe you are done, test the functionality as outlined in the "Test" section below.
  10. Continue working until the application behaves as explained in the "Test" section below.
