Get The Data
 All code for this assessment will be written only in the remote_wu.js file, no other code will be allowed with the exception of custom CSS as explained in step 8.3 below!

  1. Use the jQuery $.ajax method which begins on line 24 and currently ends on line 34.
  2. Write all code within this method, add new lines within the method as need.
  3. Send the URL to the weather underground API requesting both the geolocation and conditions objects. Use the method that allows you to send in the latitude and longitude for the current location. Don't forget to include your API key as required in the URL.
    1. The latitude (lat) and longitude (long) were gotten for you on lines 9 and 10.
  4. The dataType needed will be "jsonp".
  5. Write a success function and store its parameter as "data".
  6. I strongly encourage you to use console.log often to see what you are getting back from the API and your own variables.
  7. Base Data:
    1. You are to get the city and state and combine them into a single location name separated by a comma and space. The name is to be displayed (using a jQuery selector) in the h1 element with an id of cityDisplay AND as the first item in the "title" element within the "head" element of the page (do not replace what's there, add this in front of what's already there).
    2. You are to get the current temperature and display it into the li with an id of currentTemp. Make sure you use JavaScript to round the number to the closest integer and display the degree symbol following the number as seen in the image below.
    3. Finally, you are to display a summary of the current conditions in the li with an id of summary. Be sure the text starts with a capital letter (Hint: use the weather property from the current_observation object).
  8. Additional Data:
    1. You are to add a minimum of two additional data points to the weather page.
      1. Three list items are provided in the index page, each with an id of "add1", "add2" and "add3" for your use.
      2. Be sure that whatever you add makes sense from the site visitors perspective, don't make them guess what you meant - make it clear and obvious!
      3. You may alter the CSS within the user_styles.css file to style this content if needed (this file is already set to be used, you don't need to do anything with the main stylesheet).
